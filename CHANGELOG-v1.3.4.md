## 🚀 核心升级：技能匹配引擎重写

### 问题
v1.3.1 的技能匹配用"关键词重叠率 + Jaccard"打分，存在两个致命问题：
1. **稀有词被低估**：`PDF`、`AI` 这种高价值关键词和 `文件`、`搜索` 这种通用词权重一样
2. **阈值难调**：命中率 + Jaccard 的组合分数没有语义含义，阈值只能靠经验猜

### 解决：BM25 粗筛 + LLM 精排（两阶段过滤）

**第一阶段：BM25 粗筛（毫秒级，零 API 调用）**
- 新增 `core/bm25.py`：经典 BM25 检索引擎
- TF-IDF 自动给稀有词高权重（`PDF` IDF=0.981 > `文件` IDF=0.470）
- 长度归一化消除文档长度偏差
- 索引缓存：技能列表不变时不重建

**第二阶段：LLM 精排（仅模糊区间触发）**
- BM25 分数 ≥ 2.0 → 直接命中（高置信，零延迟）
- BM25 分数 0.5 ~ 2.0 → LLM 精排确认（DeepSeek 判断意图是否匹配）
- BM25 分数 < 0.5 → 认为不匹配，走任务分解
